# 🎬 Relatório de Projeto: Engenharia Reversa de Software Assistida por IA

## 📝 Descrição da Atividade
Este projeto foi desenvolvido sob a premissa de **Vibe Coding/Engenharia Reversa Visual**. O objetivo central foi reconstruir a funcionalidade e a interface de um aplicativo web (Sistema de Recomendação de Filmes) sem ter acesso ao código-fonte original ou documentação técnica prévia.

A metodologia consistiu em três etapas principais:
1.  **Mapeamento:** Observação da interface de referência para identificar componentes visuais e regras de negócio.
2.  **Configuração (System Instructions):** Definição do comportamento do modelo no **Google AI Studio**, instruindo-o a atuar como um desenvolvedor Full-Stack.
3.  **Construção e Validação:** Geração iterativa de código (HTML, CSS e JS) até que a estética e o comportamento replicassem a referência.

---

## 🧠 Respostas Acadêmicas

### 1. O Impacto na Formação do Engenheiro de Software Júnior
A transição do esforço da "escrita sintática" para a "descrição lógica" redefine o papel do desenvolvedor júnior. Ele deixa de ser apenas um executor de sintaxe para se tornar um **arquiteto de intenções**. 

**Competências Indispensáveis:**
* **Engenharia de Prompt e Decomposição de Problemas (Técnica):** Saber dividir um sistema complexo em requisitos lógicos menores. Sem entender a arquitetura (ex: gestão de estado, fluxo de dados), o profissional não consegue instruir a IA de forma eficaz.
* **Pensamento Crítico e Curadoria de Código (Comportamental):** Com a IA gerando a base do código, o desenvolvedor deve atuar como um revisor sênior, auditando a saída em busca de vulnerabilidades, redundâncias ou "alucinações" lógicas, garantindo a sustentabilidade do software.

### 2. Dilema Ético: Aprendizado vs. Plágio Digital
A engenharia reversa assistida por IA cruza a linha para o plágio quando há a intenção de publicar o código mimetizado como uma criação original sem valor agregado. Enquanto ferramenta de aprendizado, ela é valiosa para entender padrões de design.

**Diretriz Ética Proposta: O Protocolo de "Acréscimo de Valor e Transparência"**
* **Transparência de Origem:** Declaração explícita de quais partes do sistema foram inspiradas em referências existentes e o papel da IA no processo.
* **Inovação Diferencial Obrigatória:** Para não ser considerado plágio, o desenvolvedor deve implementar melhorias (ex: acessibilidade, performance ou novas features) que não existiam na referência original, transformando a "cópia" em uma "evolução".

## 🚀 Detalhes do Aplicativo Reconstruído
* **Interface:** Sistema de Recomendação de Filmes com foco em UX intuitiva.
* **Stack Gerada:** HTML5, CSS3 (Responsivo) e JavaScript Vanilla para lógica de filtragem.
* **Ferramenta de IA:** Google AI Studio (Modelo Gemini).

### Resultados e Aprendizados
* **Abstração Funcional:** Tradução bem-sucedida de elementos visuais em componentes de código reutilizáveis.
* **Mitigação da Paralisia de Escolha:** Implementação de lógica para sugestão de títulos baseada em preferências de gênero.
* **Vibe Coding:** Experiência prática em guiar a IA para manter a fidelidade estética da referência original.
