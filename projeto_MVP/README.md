# 🎬 CineFlow: Engine de Recomendação Inteligente

## 📝 Descrição do Projeto
O **CineFlow** é a evolução de um motor de recomendação inteligente, transformado em um produto autoral através de técnicas de **Vibe Coding** e engenharia reversa assistida por IA. O projeto migrou de uma simples análise de dados para uma aplicação Full-Stack funcional, focada em mitigar a "paralisia de escolha" com uma experiência de usuário (UX) fluida e moderna.

Reconstruído no **Google AI Studio** e potencializado pelo ecossistema **Firebase**, o CineFlow utiliza uma arquitetura híbrida para prever preferências cinematográficas e oferecer uma interface personalizada em tempo real.

## 🎨 Branding e Design
* **Novo Nome:** CineFlow
* **Estética:** Design **Glassmorfismo Minimalista**, com transparências e profundidade visual para uma experiência imersiva.
* **Componentes:** Formas orgânicas geradas via *Blobmaker* para áreas de destaque e botões com efeitos de vidro.

---

## 🚀 Diferenciais Competitivos (Novos Recursos)
Para superar a referência original, implementamos 4 funcionalidades exclusivas:
1.  **Perfil Persistente (Firebase Auth):** Sistema de login para salvar preferências e histórico de recomendações.
2.  **Lista de Desejos "Watchlist" (Cloud Firestore):** Armazenamento em tempo real de filmes que o usuário deseja assistir.
3.  **Análise de Sentimento de Reviews:** Integração com IA para classificar críticas de usuários como positivas ou negativas.
4.  **Exportação de Perfil de Gosto (PDF/JSON):** Gerador de relatórios sobre o comportamento de consumo do usuário para compartilhamento.

---

## 🔧 Stack Tecnológica & Infraestrutura
O projeto utiliza o **Firebase (Plano Spark)** para garantir escalabilidade:
* **Authentication:** Gestão de usuários.
* **Cloud Firestore:** Banco de dados NoSQL para salvar o histórico e perfis.
* **Hosting:** Publicação da ferramenta em domínio seguro (HTTPS).
* **Remote Config:** Ajuste dinâmico de cores da interface sem novo deploy.

## 🧠 Reflexão Acadêmica e Ética

### Questão A: O Engenheiro de Software na Era da IA
A mudança do foco da sintaxe para a lógica funcional exige um novo conjunto de habilidades do engenheiro júnior. Para não ser obsoleto perante modelos como o Gemini, prescrevo:
1.  **Capacidade de Abstração e Arquitetura (Técnica):** O desenvolvedor deve entender *como* os sistemas se conectam (APIs, bancos de dados, estados), pois a IA executa o código, mas o humano deve projetar o fluxo lógico.
2.  **Pensamento Analítico e Auditoria de Segurança (Comportamental):** É indispensável saber validar se o código gerado pela IA é seguro e performático, assumindo um papel de "curador técnico".

### Questão B: Originalidade vs. Plágio Digital
1.  **O Ponto de Ruptura:** A engenharia reversa torna-se plágio quando o desenvolvedor apenas "copia e cola" a solução original com o intuito de se passar pelo autor ou lucrar sem adicionar valor. Ela é aprendizado enquanto serve de base para entender padrões.
2.  **Diretriz Ética (A Regra dos 40%):** Proponho que toda solução gerada via engenharia reversa deva obrigatoriamente conter, no mínimo, 40% de funcionalidades novas ou melhorias arquiteturais significativas, acompanhadas de uma declaração de transparência sobre o uso de IA e as fontes de inspiração.
